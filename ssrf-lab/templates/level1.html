Takeaway

A server-side request primitive is not limited to remote http(s) endpoints. Depending on how the backend resolves URLs and which schemes it allows, the same code path can read local resources on the machine that performs the fetch—for example via file:// URIs or other scheme handlers the client stack understands.

In production, treat user-controlled URLs as a trust boundary: enforce strict allowlists, block dangerous schemes, use network egress controls, and separate fetchers that must reach internal assets from those that face untrusted input.
